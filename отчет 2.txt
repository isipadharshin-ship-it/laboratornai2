Лабораторная работа №1 «Разработка SPA-приложения на React»

Цель работы: Разработать одностраничное приложение (SPA) на React с использованием современных подходов к разработке веб-приложений. Изучить основы React, компонентный подход, маршрутизацию и управление состоянием.

1. Установка и настройка окружения
Для начала работы был установлен Node.js и создан новый проект с использованием Vite. Vite был выбран как современный инструмент сборки, обеспечивающий быструю разработку. Команда создания проекта: npm create vite@latest my-app -- --template react.

2. Создание базовой структуры проекта
Была создана структура папок и файлов в соответствии с лучшими практиками React-разработки. Основные директории: components, pages, context, assets. В файле main.jsx настроена точка входа приложения.

3. Реализация компонентов
Разработаны основные компоненты приложения: Header, Navigation, ThemeToggle. Каждый компонент представляет собой независимый модуль с собственной логикой отображения. Использованы функциональные компоненты с хуками.

4. Настройка маршрутизации
Установлен и настроен React Router DOM для реализации навигации между страницами без перезагрузки. Созданы маршруты для главной страницы, каталога автомобилей, корзины и панели управления.

Рисунок 1 - Структура проекта React

Вывод: В ходе лабораторной работы было разработано базовое SPA-приложение на React. Изучены основы создания компонентов, настройки маршрутизации и структуры проекта. Приобретены навыки работы с современными инструментами фронтенд-разработки.

Лабораторная работа №2 «Реализация функционала корзины и системы тем»

Цель работы: Реализовать сложную функциональность приложения включая глобальное состояние корзины, систему смены тем и взаимодействие между компонентами. Изучить Context API и управление состоянием в React.

1. Реализация Context API для управления состоянием
Созданы два контекста: CartContext для управления корзиной и ThemeContext для управления темами. В CartContext использован useReducer для обработки сложных действий с корзиной. Реализованы действия добавления, удаления и очистки корзины.

2. Система тем оформления
Реализована возможность переключения между светлой и темной темами. Тема сохраняется в localStorage для запоминания выбора пользователя. Созданы CSS-стили для обеих тем с плавными переходами.

3. Функционал корзины
Разработан компонент CartList для отображения выбранных автомобилей. Реализован расчет общей стоимости, управление количеством товаров и визуальное отображение состояния корзины. Добавлены интерактивные элементы управления.

4. Интеграция компонентов
Обеспечено взаимодействие между всеми компонентами приложения. Реализована передача данных через контексты, обновление состояния и реактивное отображение изменений. Настроены программные переходы между страницами.

Рисунок 2 - Интерфейс приложения с корзиной и темной темой

Вывод: В ходе лабораторной работы успешно реализована сложная функциональность приложения. Изучены принципы работы с глобальным состоянием, Context API и useReducer. Приобретены навыки создания интерактивных интерфейсов с сохранением состояния и тем оформления.
